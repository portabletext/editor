## What is Portable Text?

Portable Text is a JSON-based specification for structured block content. A Portable Text document is an array of typed blocks: text blocks (paragraphs, headings, lists), image blocks, code blocks, or any custom type you define. Within text blocks, you can embed inline objects (a stock ticker with symbol and exchange data) and annotations (a link that carries href, target, and tracking parameters as structured data, not just an `<a>` tag). The result is content that's portable across platforms, renderable in any format, and queryable as data.

<Tabs>
  <TabItem label="HTML string">
    ```html
    <p>Read the <a href="/docs">documentation</a> for <strong>Portable Text</strong>.</p>
    ```
  </TabItem>
  <TabItem label="Portable Text">
    ```json
    [
      {
        "_type": "block",
        "style": "normal",
        "children": [
          { "_type": "span", "text": "Read the " },
          {
            "_type": "span",
            "text": "documentation",
            "marks": ["a1b2c3"]
          },
          { "_type": "span", "text": " for " },
          {
            "_type": "span",
            "text": "Portable Text",
            "marks": ["strong"]
          },
          { "_type": "span", "text": "." }
        ],
        "markDefs": [
          {
            "_key": "a1b2c3",
            "_type": "link",
            "href": "/docs"
          }
        ]
      }
    ]
    ```
  </TabItem>
</Tabs>

Because content is structured data, you can render it as HTML, React components, Markdown, PDFs, or any other format from the same content.
